# Разбор плохого и хорошего CI/CD

## Плохие практики

1. Использование `python:latest`  
   Может неожиданно поменяться и сломать сборку.  
   В хорошем файле - фиксированная версия `python:3.12`.

2. Секреты в YAML  
   Любой участник проекта видит их в репозитории.  
   В хорошем пайплайне - секреты хранятся в GitLab CI/CD Variables.

3. Игнорирование тестов  
   `pytest || true` пропускает ошибки.  
   В хорошем пайплайне тесты - отдельный шаг и блокируют сборку.

## Результат исправлений

- Pipeline стал детерминированным.  
- Безопасность повышена.  
- Нельзя выкатить код с упавшими тестами.
